Use this endpoint to deliver branded messages to users through the Truecaller platform. Messages are displayed with your registered brand name inside the Truecaller app, giving recipients a verified, trusted experience. You can send synchronously using `instant` mode for immediate success/failure feedback, or asynchronously with a `callback` URL for high-volume sends.

## Endpoint

```
POST https://app.sling.com.ng/api/v1/send-truecaller
```

## Headers

| Header          | Value                |
| --------------- | -------------------- |
| `Accept`        | `application/json`   |
| `Authorization` | `Bearer {API_TOKEN}` |

## Parameters

<ParamField body="to" type="string" required>
  The recipient's phone number. Accepts an 11-digit local format (e.g. `08140363233`), with country code (`2348140363233`), or with a `+` prefix (`+2348140363233`).
</ParamField>

<ParamField body="message" type="string" required>
  The text content of the message. Maximum 3 pages.
</ParamField>

<ParamField body="sender" type="string" required>
  Your registered Sender ID / brand name. Must be registered on your dashboard. [Register a Sender ID](https://app.sling.com.ng/panel/sender-ids).
</ParamField>

<ParamField body="type" type="string">
  Sender ID route. Accepted values: `transactional` or `promotional`. Defaults to your Sender ID's configured route when omitted.
</ParamField>

<ParamField body="callback" type="string">
  A URL on your server that Sling will POST delivery updates to. Must be a publicly accessible HTTPS endpoint.
</ParamField>

<ParamField body="instant" type="boolean" default="false">
  When `true`, the request waits for the message to be processed and returns either a success or failure response synchronously. When `false` (default), the message is queued and the response is returned immediately.
</ParamField>

<ParamField body="contact" type="string">
  The ID of a saved contact from your Sling contact list. Use instead of `to` to send to a stored contact. [View your contacts](https://app.sling.com.ng/panel/contact).
</ParamField>

<ParamField body="group" type="string">
  The ID of a contact group. Use instead of `to` to broadcast to every contact in the group. [View your groups](https://app.sling.com.ng/panel/contact).
</ParamField>

## Request examples

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://app.sling.com.ng/api/v1/send-truecaller \
  --header 'Accept: application/json' \
  --header 'Authorization: Bearer YOUR_API_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "sender": "Brand",
    "to": "08140363233",
    "message": "Hello to Truecaller",
    "type": "transactional",
    "callback": "https://callback.io",
    "instant": false
  }'
```

```json JSON body
{
  "sender": "Brand",
  "to": "08140363233",
  "message": "Hello to Truecaller",
  "type": "transactional",
  "callback": "https://callback.io",
  "instant": false
}
```

</CodeGroup>

### URL-based request

You can also send using query string parameters for quick testing:

```
https://app.sling.com.ng/api/v1/send-truecaller?api_token=YOUR_API_TOKEN&to=08140363233&message=Hello&sender=Brand
```

<Note>
  The JSON body approach with the `Authorization` header is recommended for production. URL-based requests expose your API token in server logs and browser history.
</Note>

## Responses

### 200 — Success

<ResponseField name="status" type="string">
  Always `"success"` when the message was accepted.
</ResponseField>

<ResponseField name="message_id" type="string">
  Unique identifier for this message. Use it with the [Message Status](/api-reference/message-status) endpoint to track delivery.
</ResponseField>

<ResponseField name="sender" type="string">
  The Sender ID / brand name used to deliver the message.
</ResponseField>

<ResponseField name="message" type="string">
  The text content that was sent.
</ResponseField>

<ResponseField name="recipient" type="string">
  The normalised E.164 phone number the message was delivered to.
</ResponseField>

<ResponseField name="credit_used" type="number">
  Number of message credits deducted for this send.
</ResponseField>

```json 200 example
{
  "status": "success",
  "message_id": "2bajqzt5tw",
  "sender": "Sling",
  "message": "Hello, from Sling",
  "recipient": "2349123772222",
  "credit_used": 1
}
```

### 400 — Error

<CodeGroup>

```json Insufficient credit
{
  "status": "fail",
  "credit_used": "0",
  "details": "insufficient credit"
}
```

```json Incomplete parameters
{
  "status": "fail",
  "credit_used": "0",
  "details": "The given data was invalid"
}
```

</CodeGroup>
